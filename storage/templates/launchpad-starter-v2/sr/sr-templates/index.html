Overview

SR is a component library that is shared across projects, allowing you to build pages quickly by using predefined patterns. Each pattern is defined as a custom module and includes all CSS / JS dependencies required for the pattern to function properly.

How it Works

Every new project, once setup, has access to all custom modules of the library, which are used to create live wireframes (in HubSpot).
To add a pattern to a page, you go to Edit Page (in HubSpot) → Edit Modules → Add new module

From Wireframe to Design

The difference between a ‘wireframe page’ and a ‘designed page’ is the CSS file included in header.html
wireframe.css / .less is a trimmed down version of base.css / .less with basic color and font declarations.

Note: wireframe.css / less doesn't currently exist.

The Library

The SR Pattern Library contains 3 types of elements:

  • Components — elements such as cards or forms that can be used on their own or can be combined into a module. To allow for components to be reused these do not include containers.
  • Modules — elements such as page sections, mastheads, footers.
  • Pages — blog, blog single, landing pages, 404s.
Note: We mostly use modules and pages at the moment.
